поняття :
Стек (stack) у програмуванні — це лінійна структура
даних, що працює за принципом LIFO (Last In, First
Out — "останнім прийшов, першим вийшов"). Він діє
як стопка елементів, де додавання (push) і
видалення (pop) відбуваються лише з одного боку —
вершини. 
Стек використовується для керування :
викликами функцій, скасування дій, зворотного
виклику та аналізу виразів.
Основні поняття та застосування стека:
      Принцип роботи (LIFO): Останній
елемент, який було додано до стека, є першим, який
буде видалено.Операції: 
      Основні операції — push
(додавання на вершину) та pop (видалення з
вершини). Також використовується top/peek для
перегляду вершини. 
      Стек викликів (Call Stack):
Область пам'яті, що зберігає інформацію про активні
підпрограми (функції), локальні змінні та адреси
повернення. 
      Відмінність від Купи (Heap): Стек
автоматично керує пам'яттю (швидкий, обмежений
розмір), тоді як Купа використовується для
динамічного розподілу пам'яті.

      // У РЕАЛЬНОМУ КОДІ ПОВЕРТАТИ АДРЕСУ ЛОКАЛЬНОЇ ЗМІННОЇ

}
int main ()
{
      int a{1};
      int b;
      int c{2};
      int d{3};
      std::cout<<"Є дуже багато даних що записується в stack під час роботи програми\n"<<
      "але в цьому прикладі будуть тільки розглянуті локальні змінні,\n"<<
      "та параметри функції"<<std::endl;
      std::cout<<&a<<"    ||    "<<std::endl;
      std::cout<<&b<<"  __||__  "<<std::endl;
      std::cout<<&c<<"  \\    /  "<<std::endl;
      std::cout<<&d<<"   \\  /   "<<std::endl;
      std::cout<<e(5)<<"    \\/    "<<std::endl;
      std::cout<<"\nВ даний сегмент пам'яті\n"<<
      "дані записуються зверху вниз відносно порядку їх задання"<<std::endl;
      return 0;
}
